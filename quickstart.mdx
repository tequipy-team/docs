## Step 1: Verify your token

The simplest way to confirm your token works is to query your users list. This
is a read-only call that works with any token role.

<CodeGroup>

```bash cURL
curl -X POST https://prod.backend.tequipy.com/integration/users/query \
  -H "Authorization: Bearer $TEQUIPY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{}'
```

```python Python
import requests, os

resp = requests.post(
    "https://prod.backend.tequipy.com/integration/users/query",
    headers={"Authorization": f"Bearer {os.environ['TEQUIPY_TOKEN']}"},
    json={},
)
resp.raise_for_status()
print(resp.json())
```

```javascript Node.js
const resp = await fetch(
  "https://prod.backend.tequipy.com/integration/users/query",
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.TEQUIPY_TOKEN}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({}),
  }
);
const data = await resp.json();
console.log(data);
```

</CodeGroup>

## Step 2: Common workflows

Pick the workflow that matches your use case.

### Offboard a departing employee

When an employee leaves, create an offboarding to arrange equipment collection.
You need the employee's user ID and the IDs of the equipment to collect.

<Steps>

<Step title="Create the offboarding">

```bash cURL
curl -X POST https://prod.backend.tequipy.com/integration/offboardings \
  -H "Authorization: Bearer $TEQUIPY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "userId": "<EMPLOYEE_ID>",
    "offboardingEquipmentIds": ["<EQUIPMENT_ID>"],
    "lastWorkingDayAt": "2025-07-31T17:00:00Z",
    "pickUpDetails": {
      "fullName": "Jane Doe",
      "privateEmail": "jane.personal@example.com",
      "phoneNumber": "+49 30 12345678",
      "streetLine1": "Unter den Linden 1",
      "city": "Berlin",
      "postalCode": "10117",
      "countryCode": "DE"
    }
  }'
```

</Step>

<Step title="Track the offboarding status">

```bash cURL
curl https://prod.backend.tequipy.com/integration/offboardings/<OFFBOARDING_ID> \
  -H "Authorization: Bearer $TEQUIPY_TOKEN"
```

The `state` field progresses through:
`INITIATED` Ôćĺ `MATERIALS_SENT` Ôćĺ `COURIER_SCHEDULED` Ôćĺ `PICKED_UP` Ôćĺ `COMPLETED`

</Step>

</Steps>

### Redeploy equipment to a new hire

Send equipment from your storage to an employee's address in one API call.

<Steps>

<Step title="Create the redeployment">

```bash cURL
curl -X POST https://prod.backend.tequipy.com/integration/redeployments \
  -H "Authorization: Bearer $TEQUIPY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "employeeId": "<EMPLOYEE_ID>",
    "storageEquipmentIds": ["<USER_EQUIPMENT_ID>"],
    "startDate": "2025-08-01",
    "deliveryAddress": {
      "fullName": "Alex Smith",
      "streetLine1": "Rosenthaler Str. 40",
      "city": "Berlin",
      "postalCode": "10178",
      "countryCode": "DE"
    }
  }'
```

</Step>

<Step title="Track the delivery status">

```bash cURL
curl https://prod.backend.tequipy.com/integration/redeployments/<REDEPLOYMENT_ID> \
  -H "Authorization: Bearer $TEQUIPY_TOKEN"
```

The `state` field progresses through:
`INITIATED` Ôćĺ `PROCESSING` Ôćĺ `SENT` Ôćĺ `OUT_FOR_DELIVERY` Ôćĺ `COMPLETED`

</Step>

</Steps>
